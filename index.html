Logic Explanation

In Phase 3, I managed the application lifecycle by persisting state with localStorage. Whenever a valid mutation occurs (adding/removing loot or changing party size), I use JSON.stringify() to convert the state into a string and save it. When the page reloads, a DOMContentLoaded listener triggers restoreState(), which uses JSON.parse() to turn the string back into an object. Inside restoreState(), I use defensive programming to validate the parsed data before pushing it into my array. The updateUI() function is called strictly after restoration, ensuring the DOM is only built after the "single source of truth" is fully loaded.
