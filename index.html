Logic Explanation

In Phase 2, my loot array and party size serve as the single source of truth. I added a quantity property to the loot objects to track multiples. The updateUI() function centralizes all logic: it calculates totals by multiplying value and quantity, and re-renders the DOM every time a state change occurs (like adding/removing loot or changing party size). I use conditional logic inside updateUI() to hide the table and disable the split button when the state is invalid (like having 0 items) to protect data integrity.
